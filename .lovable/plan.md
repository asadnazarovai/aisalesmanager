
Проблема почти точно в `ChatDemo.tsx`: сейчас при любом изменении `messages` вызывается `messagesEndRef.current?.scrollIntoView(...)`. На первом рендере в `messages` уже есть стартовое сообщение бота, поэтому этот вызов срабатывает сразу при открытии страницы и браузер прокручивает весь сайт вниз до блока “Протестируйте AI Sales Manager в действии”.

Что нужно сделать:
1. Исправить автоскролл в `src/components/ChatDemo.tsx`
   - убрать `scrollIntoView` на элементе внутри страницы, потому что он скроллит не только чат, но и документ;
   - вместо этого повесить отдельный `ref` на сам контейнер сообщений (`div` с `h-80 overflow-y-auto`) и прокручивать только его внутренний скролл через `container.scrollTo({ top: container.scrollHeight, ... })`;
   - дополнительно пропустить автоскролл на самом первом рендере, чтобы сайт всегда открывался с hero-блока “AI Sales Manager”, а не прыгал вниз.

2. Сохранить корректное поведение чата
   - при отправке сообщения и при ответе бота прокручивать только окно чата вниз;
   - страница целиком не должна двигаться вообще.

3. Проверить глобальный scroll logic
   - `ScrollToTop.tsx` оставить, он сам по себе правильный и нужен для старта страницы сверху при открытии маршрута;
   - если после фикса в `ChatDemo` останется редкий прыжок, усилить `ScrollToTop` так, чтобы он один раз принудительно ставил `window.scrollTo(0, 0)` после монтирования страницы.

Технически:
- вероятный фикс: `messagesEndRef` заменить на `messagesContainerRef`;
- добавить `hasMountedRef`/`isFirstRenderRef`, чтобы не автоскроллить при первом показе стартового сообщения;
- использовать внутренний скролл контейнера чата вместо `scrollIntoView`.

Ожидаемый результат:
- открыл сайт — он начинается сверху с блока “AI Sales Manager”;
- без действий пользователя сайт сам не прокручивается;
- при тесте сообщений двигается только чат внутри карточки, а не вся страница.
